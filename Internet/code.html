OpenXP Hinweise für Entwickler

Allgemeines

OpenXP wird von einem großen Team von Entwicklern (siehe erste Seite) entwickelt. Daraus ergibt sich, das jeder Entwickler sich an ein paar Regeln halten muß, damit die Weiterentwicklung zügig ablaufen kann und so wenig wie möglich Fehlerquellen entstehen.

Zur koodinierung des Projektes existiert eine XPDeveloper Maillingliste.

Den aktuellen Code beziehen

Wir verwenden für die Verwaltung des Sourcecodes ein Versionsmanagementsystem CVS. Eine Windows und OS/2-Version des Tools befindet sich auf dem ftp-Server. Mit folgenden Befehlen kann der Code bezogen werden:

cvs -d :pserver:cvs@fries7-73.stw.uni-jena.de:/usr/local/cvs login

Als Passwort muß: "cvs" angegeben werden. Der Login ist nur einmal nötig, da CVS sich das Passwort merkt. Der Pfad nach der Option "-d" kann in der Environmentvariable "CVSROOT" abgelegt werden, damit dieser nicht bei jedem Kommando mit angegeben werden muß. Der Source selbst wird mit

cvs checkout openxp

in das Verzeichnis ./openxp abgelegt. Der Checkout erstellt eine Working Copy des Codes. Um diese Working Copy auf den aktuellsten Versionsstand zu bringen folgendes nötig:

cd openxp
cvs update

Evtl. schon durchgeführte Änderungen werden dabei nicht überschrieben, sondern mit den Neuerungen aus dem CVS Baum gemerget. Manchmal ist es nötig, eine ältere Version vom CVS-Server zu ziehen. Durch die Option -r wird ein Release Tag angegeben.

cvs up -r Beta_3_20_21


Mit diesen Befehlen wird entweder Beta 3.20.21 ausgecheckt.

OpenXP hat mittlerweile drei Entwicklerbäume, den 3.20 (16 Bit DOS), den 3.30 (16 Bit DOS) und 3.70 (32 Bit). Diese Entwicklerbäume sind unabhängig voneinander und können folgendermaßen ausgecheckt werden:

cvs up -r Branch_3_20_Release
cvs up -r Branch_3_30_Release


In der Version 3.20 sind im Prinzip keine Änderungen mehr erlaubt, außer notwendige Bugfixes. In den 3.30 und 3.70er Bäumen werden jeweils die 16 und 32 Bit Versionen weiterentwickelt.

Alle weiteren CVS-Befhele sind in der Dokumentation des Programmes bzw. in dem unten stehenden Buch erklärt.

Coding Standard

Die Code-Qualität ist für ein so großes Projekt wie OpenXP von entscheidendet Bedeutung. Weiterhin ist auf eine saubere und ausführliche Dokumentation zu achten. Änderungen, die nicht diesem Standard entsprechen, werden abgelehnt. Weiterhin sind folgende kleine Punkte zu beachten:

- Die Stringlänge befindet sich bekanntermaßen in str[0]. Bei Portieren bereitet allerdings eine Referenz Probleme, so das grundsätzlich Length(str) verwendet werden sollte
- Die Standard-Variable sollte den Typ Integer haben. Byte, Word und DWord werden nur verwendet, wenn es das Programmumfeld erfordert (z.b. Records). Integer ist auf einer 16 Bit Platform 2 Byte, auf einer 32 Bit Platform 4 Byte groß und immer das schnellste CPU Register. Die Verarbeitung von Word statt Integer ist unter 32 Bit langsamer!
- Bei dem Inline-Assembler muss darauf geachtet werden, das die Variablen SI, DI, ES ohne Probleme verwendet werden dürfen. Die Variable DS muss jedoch vorher gesichert werden.

Änderungen einbringen

Da an diesem Projekt viele Leute arbeiten, ist es wichtig einen einheitlichen Standard zu haben, wie neuer Code in die offizielle Distribution gebracht wird. Dazu wird das GNU Tool DIFF genutzt, welches auch auf unserem ftp-server liegt. Dabei ist wichtig, das der DIFF von der letzen verfügbaren Version erstellt wird, diese befindet sich ungepackt im ftp-Verzeichnis openxp/source. Um potentiellen Problemen vorzubeigen, muß DIFF mit der Option -u verwendet werden (Dank an Robo für diesen Hinweis).
