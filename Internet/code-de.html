Infos für Entwickler

Allgemeines

OpenXP wird von einem mehreren Entwicklern gleichzeitig gepflegt. Daraus ergibt sich, daß jeder Entwickler sich an ein paar Regeln halten muß, damit die Weiterentwicklung zügig ablaufen kann und so wenig Fehlerquellen wie möglich entstehen.

Zur Koordinierung des Projektes existiert die OpenXP-Dev Maillingliste.

cvsWeb kann benutzt werden, um das CVS-Repository und die Logs einzusehen (Login und Password "cvs").

OpenXP ist mit FreePascal und Virtual Pascal kompilierbar.
Die aktuellen Versionen der Compiler liegen auf dem OpenXP-ftp-Server.

Lizenztechnisches

OpenXP ist ein Open-Source-Projekt. Das heißt, daß die eingebrachten Änderungen im Rahmen der jeweiligen Lizenz ggf. ohne Rückfrage in anderen Projekten weiterverwendet werden können!

Den aktuellen Code beziehen

Verwendet wird für die Verwaltung des Sourcecodes das Versionsmanagementsystem CVS. Eine Windows und OS/2-Version des Tools befindet sich auf dem ftp-Server.

Die "Master"-Kopie der Quelltexte, die sich auf dem CVS-Server befindet, wird "Repository" genannt; die Kopie davon auf dem eigenen Rechner "Working Directory". CVS sorgt dafür, daß beide Versionen synchron gehalten werden.

Befindet sich CVS im Pfad, kann man mit folgenden Befehlen ein Working Directory anlegen:
cvs -d :pserver:username@fries7-73.stw.uni-jena.de:/usr/local/cvs login
Es wird daraufhin ein Passwort verlangt; ein Gastzugang ist als User "cvs" mit Passwort "cvs" möglich. Das Passwort wird in ~/.cvspass gespeichert (bei nicht-Unix-Systemen muß vorher mit "set home=directory" ein Homeverzeichnis festgelegt werden) und muß nach dem Login nicht mehr angegeben werden.
Mit
cvs -d :pserver:username@fries7-73.stw.uni-jena.de:/usr/local/cvs checkout openxp
legt man schließlich in "./openxp" das Working Directory an. Im Working Directory kann nach Herzenslust herumgespielt werden. Der längliche Parameter -d muß ab jetzt nicht mehr angegeben werden, da CVS den CVS-Server in den CVS-Unterverzeichnissen des Working Directories vermerkt.

Um das Working Directory auf den Stand des Repositories zu bringen, kann man ein
cvs update
machen. Evtl. schon durchgeführte lokale Änderungen werden dabei nicht überschrieben sondern mit den Neuerungen aus dem Repository ergänzt.

Um eigene Änderungen in das Repository einzubringen, braucht man einen CVS-Account mit Schreibrechten (der Gastzugang reicht dafür also nicht ;-). Dann kann man ein
cvs commit file.
machen; daraufhin wird eine Logmessage von CVS verlangt.
